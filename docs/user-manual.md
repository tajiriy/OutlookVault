## 概要

OutlookArchiver は Outlook のメールをローカルの SQLite データベースにアーカイブするデスクトップアプリケーションです。取り込んだメールは Outlook を起動しなくても閲覧・検索できます。

### 動作要件

- Windows 10 以降
- .NET Framework 4.6.2
- Microsoft Outlook（取り込み時に起動している必要があります）

## 画面構成

```
+----------------------------------------------------------+
| メニューバー [ファイル][取り込み][検索][設定][ヘルプ]       |
+----------------------------------------------------------+
| ツールバー: [今すぐ取り込み] [自動: ▶/●] [検索ボックス] [検索] |
+----------------------------------------------------------+
| フォルダツリー   | メール一覧                              |
|                  |  添付 | 件名 | 差出人 | 受信日時 | サイズ |
|  すべて (123)    |------------------------------------------|
|  受信トレイ (80) | メールプレビュー                          |
|  送信済み (43)   |  [通常表示] [会話ビュー]                  |
+----------------------------------------------------------+
| ステータスバー: 総数 123件 | 最終取り込み: 2026/03/18 12:00 |
+----------------------------------------------------------+
```

### フォルダツリー（左パネル）

- **すべて**: 全フォルダのメールを表示
- **各フォルダ**: フォルダごとのメールを表示
- フォルダ名の後の数字はアーカイブ済みメール件数

### メール一覧（右上パネル）

| 列 | 内容 |
|----|------|
| 添付 | 添付ファイルがある場合にクリップアイコンを表示 |
| 件名 | メールの件名 |
| 差出人 | 差出人の名前またはメールアドレス |
| 受信日時 | `yyyy/MM/dd HH:mm` 形式 |
| サイズ | KB または MB 単位 |

### ステータスバー

- **左側**: アーカイブ済みメール件数。Outlook に未取り込みメールがある場合は `80/123件` のように表示
- **右側**: 最終取り込み日時

## メールの取り込み

### 手動取り込み

1. ツールバーの **[今すぐ取り込み]** ボタンをクリック（または **F5** キー）
2. 設定で指定した対象フォルダから順にメールを取り込みます
3. 取り込み中はステータスバーに進捗が表示されます
4. 完了時に結果ダイアログが表示されます（設定で非表示にできます）

### 自動取り込み

1. ツールバーの **[自動: ▶]** ボタンをクリックすると自動取り込みが開始されます
2. ボタンが **[自動: ●]** に変わり、設定した間隔で定期的に取り込みが実行されます
3. もう一度クリックすると停止します

### 取り込みの仕様

- 既にアーカイブ済みのメールは自動的にスキップされます
- 添付ファイルは設定で指定したディレクトリに保存されます
- 取り込み順序（古い順/新しい順）は設定で変更できます
- 1回あたりの最大取り込み件数は設定で制限できます

## メール一覧の操作

### ソート

列ヘッダーをクリックすると、その列で昇順/降順にソートできます。現在のソート列にはヘッダーに `↑`（昇順）または `↓`（降順）が表示されます。

### 列の操作

- **列幅の変更**: 列ヘッダーの境界をドラッグ
- **列の並び替え**: 列ヘッダーをドラッグ&ドロップ
- これらの設定はアプリ終了時に自動保存され、次回起動時に復元されます

## メールの閲覧

### 通常表示タブ

メール一覧でメールを選択すると、下部のプレビューエリアに以下が表示されます。

- **ヘッダー情報**: 差出人、受信日時、件名、宛先
- **本文**: HTML またはプレーンテキスト（トグルボタンで切り替え可能）
- **添付ファイル**: ファイル名とサイズの一覧。クリックで既定のアプリケーションで開きます

### 会話ビュータブ

同じスレッドに属するメールを時系列で一覧表示します。

- スレッドは `In-Reply-To` / `References` ヘッダーで判定されます
- ヘッダーがない場合は正規化された件名（`Re:` `Fw:` 等を除去）でフォールバックします
- 引用部分は自動的に除去されて表示されます（元の本文はそのまま保持）

## 検索

検索ボックスにキーワードを入力して **Enter** キーまたは **[検索]** ボタンで実行します。検索ボックスを空にして実行すると、通常のフォルダ表示に戻ります。

### 基本検索

キーワードをそのまま入力すると、件名・本文・差出人・メールアドレス・添付ファイル名を横断して部分一致検索します。

```
amazon        → 全検索対象列で「amazon」を含むメールを表示
```

### 列指定検索

`列名: 値` で特定の列のみを検索対象にできます。`:` は部分一致、`=` は完全一致です。

```
件名: amazon         → 件名に「amazon」を含むメール
件名= 請求書         → 件名が「請求書」と完全一致するメール
差出人: 田中          → 差出人名に「田中」を含むメール
```

### 利用可能な列名

日本語名と英語名のどちらでも使用できます。大文字小文字は区別しません。

| 日本語名 | 英語名 | 検索対象 |
|---------|--------|---------|
| 件名 | subject | メールの件名 |
| 本文 | body_text | メールの本文 |
| 差出人 | sender_name | 差出人の名前 |
| メール | sender_email | 差出人のメールアドレス |
| 添付ファイル | file_name | 添付ファイルの名前 |
| 添付 | has_attachments | 添付ファイルの有無（`あり` / `なし`） |
| フォルダ | folder_name | フォルダ名 |

### 添付ファイルの有無で検索

```
添付: あり            → 添付ファイルがあるメール
添付: なし            → 添付ファイルがないメール
```

### AND / OR 検索

`and` と `or` で複数の条件を組み合わせられます。`and` は `or` より優先されます。

```
件名: amazon and 添付: あり       → 件名に amazon を含み、かつ添付あり
件名: amazon or 件名: google      → 件名に amazon または google を含む
件名: amazon and 差出人: 田中 or 本文: 注文
  → (件名に amazon AND 差出人に 田中) OR (本文に 注文)
```

### 演算子の前後の空白

演算子 `:` `=` の前後に空白があっても正しく認識されます。

```
件名 : amazon         → OK（件名: amazon と同じ）
件名 = 請求書         → OK（件名= 請求書 と同じ）
```

## メールの削除

### 削除方法

1. メール一覧でメールを選択
2. **Delete** キーを押す、または右クリックメニューから **[削除]** を選択
3. 確認ダイアログで **[はい]** をクリック

### 削除の仕様

- メールレコードがデータベースから削除されます
- 添付ファイルの実体もディスクから削除されます
- 削除したメールの Message-ID は記録され、再取り込み時にスキップされます
- **この操作は元に戻せません**

## 設定

メニューの **[設定]** から設定ダイアログを開きます。

### 保存先

| 項目 | 説明 |
|------|------|
| DB ファイル | SQLite データベースファイルのパス |
| 添付ファイル保存先 | 添付ファイルを保存するディレクトリ |

### 自動取り込み

| 項目 | 説明 | 既定値 |
|------|------|--------|
| 起動時に自動取り込みを開始する | アプリ起動時に自動取り込みを有効化 | オフ |
| 取り込み間隔（分） | 自動取り込みの実行間隔 | 10分 |
| 1回の最大取り込み件数 | 1回の取り込みで処理する最大メール数 | 500件 |
| 取り込み順序 | 古い順（推奨）/ 新しい順 | 古い順 |

### 対象フォルダ

取り込み対象の Outlook フォルダを管理します。**[追加]** でフォルダ名を入力し、**[削除]** で選択中のフォルダを除外します。

### 表示設定

| 項目 | 説明 | 既定値 |
|------|------|--------|
| デフォルトで HTML 表示する | メールプレビューの初期表示形式 | オン |
| 会話ビューを古い順で表示する | 会話ビューのソート順 | オフ |
| 取り込み完了時に結果ダイアログを表示する | 取り込み結果の表示（エラー時は常に表示） | オン |

### データ管理

**[データ初期化]** ボタンで、データベースファイルと添付ファイルディレクトリを一括削除できます。確認ダイアログが表示されます。初期化後はアプリが自動的にリセットされます（再起動不要）。

## テーブルビューア（デバッグ用）

メニューの **[ヘルプ] → [テーブル]** から開きます。データベースの生データを直接確認できるデバッグ用ツールです。

### 表示可能なテーブル

| テーブル名 | 内容 |
|-----------|------|
| emails | アーカイブ済みメール |
| attachments | 添付ファイル情報 |
| deleted_message_ids | 削除済み Message-ID |
| exchange_address_cache | Exchange アドレスキャッシュ |

### フィルタ機能

テーブルビューアのフィルタは DB 列名を使用します。

```
subject: amazon             → subject 列で部分一致
id= 1                      → id 列で完全一致
subject: amazon and sender_name: john   → AND 検索
id= 1 or id= 2             → OR 検索
```
