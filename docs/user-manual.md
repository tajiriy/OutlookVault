## 概要

OutlookVault は Outlook のメールをローカルの SQLite データベースに取り込んで保管するデスクトップアプリケーションです。取り込んだメールは Outlook を起動しなくても閲覧・検索できます。

### 動作要件

- Windows 10 以降
- .NET Framework 4.6.2
- Microsoft Outlook（取り込み時に起動している必要があります）

![画面構成](../OutlookVault/Help/screenshots/00_face.png)

### フォルダツリー（左パネル）

- **すべて**: 全フォルダのメールを表示
- **各フォルダ**: フォルダごとのメールを表示
- **ゴミ箱**: 削除済みメールを表示（復元・完全削除が可能）
- フォルダ名の後の数字はメール件数

### メール一覧（右上パネル）

| 列 | 内容 |
|----|------|

| 件名 | メールの件名 |
| 差出人 | 差出人の名前またはメールアドレス |
| 受信日時 | `yyyy/MM/dd HH:mm` 形式 |
| サイズ | KB または MB 単位 |

### ステータスバー

- **左側**: 取り込み済みメール件数。Outlook に未取り込みメールがある場合は `80/123件` のように表示
- **右側**: 最終取り込み日時

## メールの取り込み

### 手動取り込み

1. ツールバーの **[今すぐ取り込み]** ボタンをクリック（または **F5** キー）
2. 設定で指定した対象フォルダから順にメールを取り込みます
3. 取り込み中はステータスバーに進捗が表示されます
4. 完了時に結果ダイアログが表示されます（設定で非表示にできます）

### 自動取り込み

自動取り込みには2つのモードがあります。設定画面のラジオボタンで切り替えます。

#### 間隔モード（デフォルト）

1. ツールバーの **[自動: ▶]** ボタンをクリックすると自動取り込みが開始されます
2. ボタンが **[自動: ●]** に変わり、設定した間隔で定期的に取り込みが実行されます
3. もう一度クリックすると停止します

#### 定時モード

1. 設定画面で「定時取り込み」を選択し、時刻を指定します（例: 20:00）
2. ツールバーの **[自動: ▶]** をクリックすると **[定時: 20:00]** に変わります
3. 毎日指定時刻になると、最大件数制限なしで**完全同期**が実行されます
4. 同じ日に2回実行されることはありません

### 同期モード

取り込みには2つの同期モードがあります。設定画面で切り替えます。

#### フルスキャン

対象フォルダ内のすべてのメールを走査し、未取り込みのメールを取り込みます。メール数が多い場合は時間がかかります。

#### 差分スキャン（デフォルト・推奨）

前回の同期日時を基準に、それ以降に受信したメールのみを走査します。Outlook の `ReceivedTime` フィルタを使用するため、フルスキャンと比べて大幅に高速です。

- **バッファ時間**: 前回同期日時から指定時間分さかのぼってスキャンします（デフォルト: 24時間）。万が一の取りこぼしを防ぐための安全マージンです
- **自動フォールバック**: 完全同期（フルスキャン）が1回も完了していないフォルダでは、差分スキャンを選択していても自動的にフルスキャンが実行されます。これにより、過去のメールが漏れることを防ぎます
- **定時モード**: 定時取り込みは常にフルスキャンで実行されます（同期モード設定に関わらず）

### 取り込みの仕様

- 既に取り込み済みのメールは自動的にスキップされます
- 添付ファイルは設定で指定したディレクトリに保存されます
- 取り込み順序（古い順/新しい順）は設定で変更できます
- 1回あたりの最大取り込み件数は設定で制限できます
- **エラーになったメールは自動的に除外リストに登録され、次回以降の取り込みでスキップされます**（詳細は「エラー除外リスト」を参照）

### エラー発生時の表示

取り込み中にエラーが発生した場合、完了ダイアログにはエラーの種類別の件数サマリーが表示されます。

```
取り込み完了
取り込み: 10件 / スキップ: 5件 / エラー: 3件

エラーサマリー:
  ・アクセスが拒否されました: 2件
  ・タイムアウトしました: 1件

詳細はログファイルを参照:
C:\...\import_errors_20260318_143000.log
```

個々のエラーの詳細（日時、MessageID、件名、エラー内容）は、アプリケーション（exe）と同じフォルダにログファイル（`import_errors_yyyyMMdd_HHmmss.log`）として出力されます。

エラーになったメールは自動的に「エラー除外リスト」に登録され、次回以降の取り込みでスキップされます。エラー除外によりスキップされたメールがある場合、結果に「エラー除外: N件」と表示されます。

```
取り込み完了
取り込み: 10件 / スキップ: 5件 / エラー: 0件 / エラー除外: 3件
```

### エラー除外リスト

メニューの **[取り込み] → [エラー除外リスト]** から管理画面を開きます。

過去の取り込みでエラーになったメールの一覧が表示されます。各エントリには以下の情報が含まれます。

| 列 | 内容 |
|----|------|
| MessageID | メールの一意な識別子 |
| フォルダ | エラー発生時のフォルダ名 |
| 件名 | メールの件名 |
| エラー内容 | エラーメッセージ |
| 受信日時 | メールの受信日時 |
| 送信者 | メールの送信者名 |
| エラー発生日時 | エラーが登録された日時 |

#### 除外の解除

- **個別解除**: 一覧から項目を選択して **[選択した項目の除外を解除]** をクリック
- **全件解除**: **[すべての除外を解除]** をクリック

除外を解除すると、次回の取り込みで再度取り込みが試みられます。問題が解消していない場合は、再びエラーになり除外リストに登録されます。

### 取り込みの中止

取り込み中にメニューの **[取り込み] → [取り込み中止]** を選択すると、確認ダイアログが表示されます。「はい」を選択すると取り込みを安全に中断します。処理中のバッチはロールバックされ、中間コミット済みのデータはそのまま保存されます。

> **注意**: 取り込み中でない場合、「取り込み中止」メニューはグレーアウトされます。

### 取り込み中のアプリ終了

取り込み中にアプリケーションを閉じようとすると、確認ダイアログが表示されます。

- **「はい」を選択**: 取り込みを安全に中断してから終了します。処理中のバッチはロールバックされ、データベースが不整合な状態になることはありません。中間コミット済みのデータはそのまま保存されます。
- **「いいえ」を選択**: 終了をキャンセルし、取り込みを継続します。

## メール一覧の操作

### ソート

列ヘッダーをクリックすると、その列で昇順/降順にソートできます。現在のソート列にはヘッダーに `↑`（昇順）または `↓`（降順）が表示されます。

### 列の操作

- **列幅の変更**: 列ヘッダーの境界をドラッグ
- **列の並び替え**: 列ヘッダーをドラッグ&ドロップ
- これらの設定はアプリ終了時に自動保存され、次回起動時に復元されます

### メールを別ウィンドウで開く

メール一覧のメールを**ダブルクリック**すると、そのメールを別ウィンドウで表示できます。

- タイトルバーにメールの件名が表示されます
- HTML/テキストの切り替えボタンが使えます
- 複数のメールを同時に別ウィンドウで開くことができます
- メインウィンドウとは独立して操作・閉じることができます

## メールの閲覧

### 通常表示タブ

メール一覧でメールを選択すると、下部のプレビューエリアに以下が表示されます。

- **ヘッダー情報**: 差出人、受信日時、件名、宛先
- **本文**: HTML またはプレーンテキスト（トグルボタンで切り替え可能）
- **添付ファイル**: ファイルタイプ別の大きめアイコンとファイル名の一覧（画面下部に横並び表示）。ホバーでフルファイル名とサイズを表示、ダブルクリックで既定のアプリケーションで開きます。右クリックで「名前を付けて保存」が可能です
  - **セキュリティ**: `.exe`、`.bat`、`.vbs` 等の実行可能ファイルはダブルクリックで開けません。「名前を付けて保存」で保存してから開いてください
- **リンク**: HTML メール内のリンクをクリックすると、確認ダイアログが表示されます。「はい」を選択するとデフォルトブラウザで開きます

### 会話ビュータブ

同じスレッドに属するメールを時系列で一覧表示します。

- スレッドは `In-Reply-To` / `References` ヘッダーで判定されます
- ヘッダーがない場合は正規化された件名（`Re:` `Fw:` 等を除去）でフォールバックします
- 引用部分は自動的に除去されて表示されます（元の本文はそのまま保持）

## 検索

検索ボックスにキーワードを入力して **Enter** キーまたは **[検索]** ボタンで実行します。**[クリア]** ボタンを押すと検索条件をリセットし、通常のフォルダ表示に戻ります。検索ボックスを空にして検索を実行しても同様です。

### 基本検索

キーワードをそのまま入力すると、件名・本文・差出人・メールアドレス・添付ファイル名を横断して部分一致検索します。

```
amazon        → 全検索対象列で「amazon」を含むメールを表示
```

### 列指定検索

`列名: 値` で特定の列のみを検索対象にできます。`:` は部分一致、`=` は完全一致です。

```
件名: amazon         → 件名に「amazon」を含むメール
件名= 請求書         → 件名が「請求書」と完全一致するメール
差出人: 田中          → 差出人名に「田中」を含むメール
```

### 利用可能な列名

日本語名と英語名のどちらでも使用できます。大文字小文字は区別しません。

| 日本語名 | 英語名 | 検索対象 |
|---------|--------|---------|
| 件名 | subject | メールの件名 |
| 本文 | body_text | メールの本文 |
| 差出人 | sender_name | 差出人の名前 |
| メール | sender_email | 差出人のメールアドレス |
| 添付ファイル | file_name | 添付ファイルの名前 |
| 添付 | has_attachments | 添付ファイルの有無（`あり` / `なし`） |
| フォルダ | folder_name | フォルダ名 |

### 添付ファイルの有無で検索

```
添付: あり            → 添付ファイルがあるメール
添付: なし            → 添付ファイルがないメール
```

### AND / OR 検索

`and` と `or` で複数の条件を組み合わせられます。`and` は `or` より優先されます。

```
件名: amazon and 添付: あり       → 件名に amazon を含み、かつ添付あり
件名: amazon or 件名: google      → 件名に amazon または google を含む
件名: amazon and 差出人: 田中 or 本文: 注文
  → (件名に amazon AND 差出人に 田中) OR (本文に 注文)
```

### 演算子の前後の空白

演算子 `:` `=` の前後に空白があっても正しく認識されます。

```
件名 : amazon         → OK（件名: amazon と同じ）
件名 = 請求書         → OK（件名= 請求書 と同じ）
```

## メールの削除

### ゴミ箱に移動

1. メール一覧でメールを選択
2. **Delete** キーを押す、または右クリックメニューから **[削除]** を選択
3. 確認ダイアログで **[はい]** をクリック

削除したメールはフォルダツリーの **[ゴミ箱]** に移動します。データベースやファイルからは削除されないため、後から復元できます。

### ゴミ箱の操作

フォルダツリーで **[ゴミ箱]** を選択すると、削除済みメールの一覧が表示されます。

| 操作 | 方法 |
|------|------|
| 復元 | メールを選択して右クリック → **[復元]** |
| 完全に削除 | メールを選択して右クリック → **[完全に削除]** または **Delete** キー |
| ゴミ箱を空にする | ゴミ箱ノードを右クリック → **[ゴミ箱を空にする]** |

### 完全削除の仕様

「完全に削除」または「ゴミ箱を空にする」で実行される完全削除は以下の動作をします。

- メールレコードがデータベースから削除されます
- 添付ファイルの実体もディスクから削除されます
- 削除したメールの Message-ID は記録され、再取り込み時にスキップされます
- **この操作は元に戻せません**

### ゴミ箱の自動削除

ゴミ箱に入ってから一定日数が経過したメールは、アプリ起動時に自動的に完全削除されます。

- デフォルト: 30日
- 設定で日数を変更できます（0 で自動削除を無効化）

### 削除同期（Outlook 側の削除を反映）

Outlook 側でメールを削除した場合、設定で有効にすると、取り込み実行時にゴミ箱に自動的に移動されます。

1. **設定 → 自動取り込み → 「Outlook 側で削除されたメールをデータベースからも削除する」** にチェック
2. 取り込み（手動/自動）を実行すると、通常の取り込み処理の後に削除同期が行われます
3. 結果ダイアログに「削除同期: N件」と表示されます

#### 注意事項

- 対象フォルダ内の全メールの Message-ID を Outlook から取得するため、メール数が多い場合は処理時間がかかります
- Outlook でメールを別フォルダに移動した場合、元のフォルダからは削除扱いになります（移動先フォルダが対象フォルダに含まれていれば、次回取り込み時に再取り込みされます）
- この設定はデフォルトでオフです

## 自動削除ルール

メールの取り込み時に、指定した条件にマッチするメールを自動でゴミ箱に移動する機能です。メニューの **[取り込み] → [自動削除ルール]** から管理画面を開きます。

### ルールの作成

1. **[追加]** ボタンをクリック
2. ルール名を入力（例: 「広告メール削除」）
3. フィルタ式を入力（検索ボックスと同じ構文）
4. **[プレビュー]** ボタンで現在マッチするメール件数を確認
5. **[OK]** で保存

### フィルタ式の例

```
差出人: noreply               → 差出人に noreply を含むメール
件名: 広告 AND 添付: なし      → 件名に「広告」を含み添付なしのメール
メール: @marketing.example.com → メールアドレスに特定ドメインを含むメール
```

検索機能と同じ構文（列指定、AND/OR、完全一致/部分一致）がすべて使えます。

### ルールの管理

| 操作 | 説明 |
|------|------|
| 追加 | 新しいルールを作成 |
| 編集 | 選択したルールの名前・フィルタ式を変更 |
| 削除 | 選択したルールを削除 |
| 有効/無効 | ルールの有効・無効を切り替え |
| 再適用 | 有効なルールを既存メールに再適用（マッチしたメールをゴミ箱に移動） |

### 適用タイミング

- **取り込み時**: メールの取り込み完了後、有効なルールが自動的に適用されます。マッチしたメールはゴミ箱に移動し、取り込み結果に「自動削除: N件」と表示されます
- **再適用**: ルール管理画面の **[再適用]** ボタンで、既存の全メールに対してルールを再適用できます

### 安全性

- 自動削除はゴミ箱への移動（論理削除）です。誤ったルールで削除しても、ゴミ箱から復元できます
- ルール作成時に **[プレビュー]** で事前にマッチ件数を確認することを推奨します

## タスクトレイ常駐

OutlookVault はタスクトレイ（通知領域）に常駐し、バックグラウンドで定期的にメールを取り込むことができます。

### Windows 起動時の自動起動

設定で「Windows 起動時に自動起動する」を有効にすると、Windows ログイン時にアプリがタスクトレイに常駐した状態で自動起動します。ウィンドウは表示されず、トレイアイコンのみが表示されます。管理者権限は不要です。

### トレイへの格納

以下の操作でアプリをタスクトレイに格納できます。

- **最小化**: ウィンドウを最小化すると自動的にトレイに格納されます（設定で無効化可能）
- **閉じるボタン**: ウィンドウの × ボタンをクリックするとトレイに格納されます（設定で無効化可能）

トレイに格納中もアプリは動作を続け、自動取り込みが有効な場合は定期的にメールを取り込みます。

### トレイからの復帰

- トレイアイコンを**ダブルクリック**するとウィンドウが表示されます
- トレイアイコンを**右クリック**してメニューから **[表示]** を選択しても復帰できます

### トレイアイコンの右クリックメニュー

| メニュー | 操作 |
|---------|------|
| 表示 | ウィンドウを表示（太字 = デフォルト操作） |
| 今すぐ取り込み | ウィンドウを表示して取り込みを開始 |
| 終了 | アプリケーションを完全に終了 |

### バルーン通知

トレイ常駐中に取り込みが完了すると、バルーン通知で結果が表示されます（設定で無効化可能）。

```
取り込み完了
取り込み: 15件 / スキップ: 120件
```

エラーが発生した場合は警告アイコンで表示されます。

### アプリの完全終了

トレイ常駐中にアプリを完全に終了するには、以下のいずれかの方法を使用してください。

- トレイアイコンを右クリック → **[終了]**
- メニューの **[ファイル] → [終了]**
- キーボードの **Alt+F4**（ウィンドウ表示中）

## 設定

メニューの **[設定]** から設定ダイアログを開きます。

### 保存先

| 項目 | 説明 |
|------|------|
| DB ファイル | SQLite データベースファイルのパス |
| 添付ファイル保存先 | 添付ファイルを保存するディレクトリ |

### 自動取り込み

| 項目 | 説明 | 既定値 |
|------|------|--------|
| 起動時に自動取り込みを開始する | アプリ起動時に自動取り込みを有効化 | オフ |
| 取り込み間隔（分） | 間隔モード: N分ごとに取り込みを実行 | 10分 |
| 定時取り込み | 定時モード: 毎日指定時刻に完全同期を実行（件数制限なし） | 20:00 |
| 1回の最大取り込み件数 | 間隔モードで処理する最大メール数（定時モードでは無制限） | 100件 |
| 取り込み順序 | 古い順（推奨）/ 新しい順 | 古い順 |
| 同期モード | フルスキャン / 差分スキャン（推奨） | 差分スキャン |
| 差分スキャンのバッファ | 前回同期日時からさかのぼるバッファ時間（時間単位） | 24時間 |
| Outlook 側で削除されたメールをデータベースからも削除する | 取り込み時に Outlook から消えたメールをデータベースから削除 | オフ |

### 対象フォルダ

取り込み対象の Outlook フォルダを管理します。**[選択...]** ボタンをクリックすると、Outlook に接続してメールフォルダの一覧を取得し、チェックボックスで対象フォルダを選択できます。フィルタ欄にテキストを入力するとフォルダ名で絞り込めます。

> **注意**: カレンダー・連絡先・Yammer・会話の履歴などメール以外のフォルダは自動的に除外され、メールフォルダのみが一覧に表示されます。

### 表示設定

| 項目 | 説明 | 既定値 |
|------|------|--------|
| デフォルトで HTML 表示する | メールプレビューの初期表示形式 | オン |
| 会話ビューを古い順で表示する | 会話ビューのソート順 | オン |
| 取り込み完了時に結果ダイアログを表示する | 正常完了時に取り込み結果を表示 | オン |
| 取り込みエラー時に結果ダイアログを表示する | エラー発生時に取り込み結果を表示 | オン |

### タスクトレイ

| 項目 | 説明 | 既定値 |
|------|------|--------|
| Windows 起動時に自動起動する（トレイに常駐） | ログイン時にトレイ常駐で自動起動 | オフ |
| 最小化時にタスクトレイに格納する | ウィンドウ最小化でトレイに格納 | オフ |

### データ管理

**[データ初期化]** ボタンで、データベースファイルと添付ファイルディレクトリを一括削除できます。確認ダイアログが表示されます。初期化後はアプリが自動的にリセットされます（再起動不要）。

## テーブルビューア（デバッグ用）

メニューの **[開発] → [テーブルビューア]** から開きます。データベースの生データを直接確認できるデバッグ用ツールです。

### 表示可能なテーブル

| テーブル名 | 内容 |
|-----------|------|
| emails | 取り込み済みメール |
| attachments | 添付ファイル情報 |
| deleted_message_ids | 削除済み Message-ID |
| exchange_address_cache | Exchange アドレスキャッシュ |
| error_message_ids | エラー除外 Message-ID |
| folder_sync_state | フォルダごとの同期状態 |

### フィルタ機能

テーブルビューアのフィルタは DB 列名を使用します。

```
subject: amazon             → subject 列で部分一致
id= 1                      → id 列で完全一致
subject: amazon and sender_name: john   → AND 検索
id= 1 or id= 2             → OR 検索
```

### セルのコピー

テーブルビューアではセル単位で選択・コピーができます。

- **単一セル**: クリックで選択し、**Ctrl+C** または右クリックメニューの **[コピー]** でクリップボードにコピー
- **複数セル**: Ctrl+クリック や Shift+クリック で複数セルを選択可能
- **矩形選択**（連続した行×列）: 横はタブ区切り、縦は改行区切りでコピー
- **とびとび選択**: タブ区切りでフラット出力

コピーした値をフィルタ欄に貼り付けて、そのままフィルタ検索に利用できます。
